Sharing in ODS-Calendar

ODS-Calendar allows users to share their calendars with other users in the relevant ODS data space. Here is a sample scenario:

  1. Log in as the demo user, and go to the Calendar.



  2. Go to Preferences.



  3. Click Share my Calendar.
  4. The Sharing to user(s) form will open.



  5. Click the Select button to choose ODS user(s), or enter them directly in the "User(s)" field.
  6. If you click the Select button, a new window will open with a list of all registered users in this ODS instance. You can select users directly, or make a filter. In our example, we will select the test1 user.



  7. You can enable or disable sharing for a selected user by ticking (or un-ticking) the Enable check-box, and set the Share mode to Read Only or Read/Write.



  8. Click the Share button to save your settings.
  9. Now you'll see a list of all users to whom your calendar has been shared, including the new user(s). In our case, this is just user test1.



  10. Now let's see what user test1 has in their Calendar. Log out as user demo, and log in as user test1. Then, go to Calendar.



  11. Go to Preferences -> Accepted Calendars to see all calendars to which you've already subscribed (i.e., accepted).



  12. Click the Accept button to review those for which sharing has been offered, but which you have not yet accepted (i.e., subscribed to).
  13. For "Calendar type", choose Shared to me to select from calendars shared specifically with you, or Public to select from Public Calendars (i.e., calendars shared with all ODS users); then click the "Select Shared" button.



  14. In the pop-up window, select the demo user's calendar, "mycalendar".



  15. The "My Accepted Calendars/Create" form will now have "mycalendar" in the Calendar name field. Tasks and other items from this calendar will be shown in the "Color" you set here.



  16. Click the Accept button, and the Accepted Calendars list will be updated to reflect your changes.



  17. Now user test1's calendar also shows the shared tasks and events from user demo's calendar. Because demo shared their calendar as Read Only, items from that calendar will not be editable; i.e.,
