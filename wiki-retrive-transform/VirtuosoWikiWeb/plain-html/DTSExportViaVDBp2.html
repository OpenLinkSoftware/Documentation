Part II: Prepare & Test your ODBC Data Sources

1. Launch the ODBC Data Sources Administrator which usually resides in the Administrative Tools folder in the Control Panel.

2. Click the System Administrator tab.

3. Locate or create a native SQL Server Data Source Name that connects to your Source database.

4. Test your DSN.

5. Locate or create a Data Source Name that connects to your Source database.

6. Test your DSN.

7. Close the ODBC Data Sources Administrator.
