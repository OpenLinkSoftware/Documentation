==Receive Size==

Receive Size is a widespread parameter which appears in each of <nop>OpenLink's configuration files and utilities. Receive Size may be set on both the client and server. It represents the size of a component's RPC receive buffer. (This is the amount of space which the component may use to store incoming messages.) Furthermore, your client and server Receive Size values should be identical. However, you do not want to alter these parameters without guidance from <nop>OpenLink Product Support. 

See Also: Receive Buffer, Send Size
