Admin User Account is not registered

If you are using Microsoft Access and you receive "Admin User Account is not registered" error messages, take the following action:

  1. Use a text editor to open the oplrqb.ini file which you find in the /bin/ sub-directory of your OpenLink server components installation.
  2. Locate the [Mapping Rules] section.
  3. Place a ";" in front of the following line: 


    *:*:*:*:*:msaccess:* = reject Admin user account is not registered

    For example: 


    ;*:*:*:*:*:msaccess:* = reject Admin user account is not registered

  4. Restart your Broker.
  5. Test your connection.
