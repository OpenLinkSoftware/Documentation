Host:Port

Host:Port is the convention used to reference both the server and TCP port on which a Multi-Tier Request Broker runs. Host passes the IP address or canonical name of the server. Port passes the Listen port, which is defined in the Session Rules Book. The Session Rules Book is the oplrqb.ini file that appears in the /bin sub-directory of the OpenLink server components installation.

Note: Host:Port appeared in OpenLink 4.x builds. OpenLink 1.x and 3.x did not have this convention. OpenLInk 1.x and 3.x passed a Host value and assumed the Request Broker was running on TCP port 5000.
