Error Message: Server/Workstation requirements not licensed

The "[OpenLink][ODBC]Server/Workstation requirements not licensed" error message typically indicates that a Workstation-class license has been deployed on a Server-class OS.

(The reverse is sometimes but rarely true. Licenses permitting use on Server-class OS are intended to always also permit use on Workstation-class OS, but errors sometimes occur in license generation.)

Our Support or Sales Team can provide you with a temporary Server-class license to confirm that this is the problem. Once confirmed, we can help you purchase any appropriate upgrade.

Please note that "Express" Edition drivers are generally licensed for use only on Workstation-class OS. For most deployments on Server-class OS, you need to use a "Lite" or "Enterprise" Edition driver.

Evidence

  • License file (e.g., virtuoso.lic, oplrqb.lic, sql_lt.lic, ee_inf_lt.lic)
  • Complete Operating System name and version
