Core Dump Files


OpenLink Product Support staff do not interpret core files. Neverthless, core files should not be ignored. They may be indicative of serious problems. Development may actually want to view them, if they feel the cores signal a product flaw.

If you receive a core file, take the following action:

1. Ascertain the sequence of events that produced the core.

2. Forward the core file to OpenLink Product Support.

3. Forward the full text of any errors associated with the core to OpenLink Product Support.

4. Provide the uname -a output and any other patch level information about the machine on which the core was produced.

5. Provide version information for all OpenLink products associated with the core.

OpenLink Product Support will work to reproduce the core in-house. If they succeed, they will open a Bugzilla and attach the core file to that Bugzilla. They will also provide OpenLink Development with all details of the test environment so that they can reproduce and resolve the problem themselves.
