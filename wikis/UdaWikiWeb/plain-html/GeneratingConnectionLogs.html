Generating Connection Logs

Use the following instructions to generate Multi-Tier connection logs. These logs are generally referred to as Request Broker log files.


Command Line Method (Requires Shutdown)

  1. Open an MS-DOS command prompt or Unix terminal.
  2. cd into the /bin sub-directory of your OpenLink server components installation.
  3. Execute this command: oplshut -fy
  4. Execute this command: oplrqb +loglevel 7 +logfile oplrqb.log
  5. Recreate the error condition.
  6. Retrieve the log and forward it to Product Support.

Web-based Admin Assistant (No Shutdown Required)

  1. Open your Admin Assistant in a Web browser.
  2. Select "Server Components Administration."
  3. Select "Request Broker Administration."
  4. Select "Edit Request Broker's Parameters (Wizard)."
  5. Enter your Admin Assistant username & password. (Both values default to "admin".)
  6. Advance one screen and set "Log Detail" to "DEBUG."
  7. Scroll through the screens to end.
  8. Hit "Save."
  9. Click "Reinitialize OpenLink Request Broker."
  10. Recreate the error condition.
  11. Retrieve the log and forward it to Product Support.
