License Has Expired

  1. Log in to the target operating system.
  2. Stop the OpenLink License Manager Service (oplmgr) associated with Release 5.x and 6.x installations.
  3. Search the machine to ensure that all expired license files are removed.
  4. Locate the correct license file and put it in place.
  5. Set OPL_LICENSE_DIR to point to the directory containing the license.
  6. Ensure there are no permissions issues associated with the license.
  7. Restart the OpenLink License Manager.
  8. Restart the Multi-Tier Request Broker service, if applicable.
  9. Restart the HTTP-based iODBC Data Sources Administrator, if applicable.
  10. Restart any/all ODBC client applications, including any which connect through an OLE DB Provider for ODBC or an ADO.NET Provider for ODBC
  11. Test.

If the problem persists, have OpenLink Product Support check the integrity of the license.
