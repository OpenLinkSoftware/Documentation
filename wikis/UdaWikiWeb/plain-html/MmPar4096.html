Server has -Mm parm 4096 and client has 1024. They must match. (1150)

-Mm is a message size parameter. It is being passed by the client and server. Moreover, as the message indicates, the values which the parameters take are different. This cannot be. You must change one to match the other. For best results, try pass -Mm 2048 in your startup.pf files. Then, double-click on the .pf files to initialize the changes. If the problem persists, you may need to pass the -Mm parameter as an OpenLink Server Options. Contact OpenLink Product Support for additional information.
