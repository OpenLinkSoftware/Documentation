Retry Timeout

[1] The Retry Timeout (RetryTimeout) parameter appears in the [Communications] section of the OpenLink Session Rules Book. This parameter represents the number of seconds between the Request Broker's attempts to contact a database agent which did not respond previously.

[2] The Retry parameter appears in the udbc.ini file, the OpenLink Client Configurator, and the Administrative Assistant's Client Networking Parameters box. Here, the parameter specifies the number of seconds which elapse between a client's attempts to reach the Broker.
