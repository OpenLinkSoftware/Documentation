When do I need the Microsoft SQL Server native client?

OpenLink's current data access products do not use the Microsoft SQL Server native client. Use of the Microsoft SQL Server native client was deprecated with the advent of OpenLink 4.0.

There are two scenarios in which you would require the Microsoft SQL Server native client. You must install the Microsoft SQL Server client on the same machine as any 1.x or 3.x Single-Tier driver. Additionally, you must install the Microsoft SQL Server client on the same machine as OpenLink's Request Broker and Microsoft SQL Server Database Agent, if the Broker and Microsoft SQL Server Agent do not reside on the same machine as the Microsoft SQL Server database, and the Microsoft SQL Server Agent is version 1.x or 3.x.
