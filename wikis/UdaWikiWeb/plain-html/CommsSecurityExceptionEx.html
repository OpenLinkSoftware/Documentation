com.ms.SecurityExceptionEx[openlink/jdbc/OPLRPCClient.ClientConnect]: cannot connect to "<server>"

OpenLink's JDBC applets to query databases on the server from which the applet was called. For instance, if you call your applet from http://localhost:8000, you may only query databases on localhost. If you attempt to connect to databases on a different server, you will encounter "SecurityExceptionEx" errors.

If you experience SecurityExceptionEx errors, you may be able to use OpenLink's JDBC applications. You will find the JDBC applications in the OpenLink Data Access Drivers folder. This folder is linked to the Windows' Programs menu.
