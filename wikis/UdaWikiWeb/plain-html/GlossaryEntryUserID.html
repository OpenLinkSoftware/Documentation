User_ID

The User_ID parameter appears in the [generic_xxx] sections of the OpenLink Session Rules Book. You may use this parameter to hardcode user names. However, hardcoded names will override all names passed in a client DSN. If OPSYSLOGIN is enabled, the operating system will authenticate the User_ID. If OPSYSLOGIN is disabled, User_ID will be authenticated by the backend database.

Example:

User_ID = scott
