#DELETED

Microsoft Access may display cell values as #Deleted when there is a primary key conflict in the result set and/or linked table, such that Access cannot uniquely identify one or more individual records.

This is seen relatively commonly when queries are issued against heterogeneous tables (i.e., tables from multiple remote data sources) which have been linked into Access.
