Gathering Additional Evidence: Dumping Progress/OpenEdge? Data

The following instructions will enable you to dump data from your Progress tables.





  • Progress Servers on Unix-like OS

  • Progress Servers on Windows



Progress Servers on Unix-like OS

  1. Log in to the server as the Progress administrative user.
  2. Run the progress user's .profile.
  3. Navigate to the $DLC/bin directory.
  4. Execute the following command: 


    _progres

  5. If prompted, hit the space bar to continue.
  6. Practice navigating the interface. (Use F3 to navigate the menus. The down arrow keys expand menus. Tab jumps you around from subject to subject.)
  7. Select Tools >> Data Dictionary.
  8. Select Database >> Connect and input the path to your database. Tab will take you to <files> which allows you to browse the file system for your database.
  9. Select Admin >> Dump Data & Definitions >> Table Contents.
  10. Select that table that you want to dump.
  11. Click F1.
  12. Click OK.
  13. Locate your new .d file.
  14. ZIP the file, and email the archive to technical.support@openlinksw.com, referencing your case ID in the subject header.

Progress Servers on Windows

  1. Open your Progress Data Administration utility.
  2. Expand the Database drop-down menu.
  3. Select Connect.
  4. Specify the database to which you want to connect.
  5. Click on OK.
  6. Expand the Admin drop-down menu.
  7. Select Dump Data and Definitions >> Table Contents.
  8. Highlight the table from which you want to dump.
  9. Click OK.
  10. Take note of the Output File.
  11. Click OK.
  12. Exit the Data Administration utility.
  13. Locate your new .d file.
  14. ZIP the file, and email the archive to technical.support@openlinksw.com, referencing your case ID in the subject header.
