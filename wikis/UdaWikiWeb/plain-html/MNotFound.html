^M: Not found

All downloaded files should be transferred with binary-preserving tools. If using FTP, OpenLink's Request Broker and Database Agent archives (.tgz files) must be transferred in BINARY mode, while the installation shell scripts (.sh files) may be transferred with either TEXT or BINARY mode. However -- if once transferred by TEXT, they must always be transferred as TEXT, to maintain the cross-platform line-end translations.

If you mixed TEXT and BINARY modes while transferring a shell script, you may receive ^M errors. To remedy this problem, re-download, and transfer consistently.

Evidence

  • Output of uname -a
  • Installation Shell Script, install.sh
