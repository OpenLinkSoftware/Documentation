==Trace==

[1] Traces capture debug output produced by a variety of programs. For instance, clientside ODBC traces record the series of ODBC function calls made by an application. Broker traces depict events which transpire when the Broker resolves a connection request. You may be asked to supply one or more specific traces, if you encounter difficulties with <nop>OpenLink software. Moreover, trace instructions will be provided, at that time.   

[2] <nop>OpenLink's Unix-based odbc.ini file contains a Trace parameter. If the Trace parameter is set to 0, no tracing will occur. If the Trace parameter is set to 1, ODBC function calls will be written to the file specified by the odbc.ini traceFile parameter.   

See Also: Debug File, Trace File
