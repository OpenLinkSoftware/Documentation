
= doc.FAQ.WhatFactorsContributeToStabilityProblems

%TOC%

   
*Improper use of database agent reuse rules in the Multi-Tier oplrqb.ini file  
*Inadequate memory  
*Inadequate processor count  
*Insufficient allocation of TCP ports in the Multi-Tier oplrqb.ini file  
*Lack of critical database patches or application of flawed patches  
*Lack of critical o/s patches or application of flawed patches  
*Sub-optimal kernel tuning parameters (Unix)  
*Use of deprecated [[OpenLink]]  components  
*Use of [[OpenLink]]  products with an incompatible databases or o/s  
*Use of Single-Tier drivers on top of unstable database native clients
