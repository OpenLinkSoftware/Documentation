---+WebID ACL protect a mounted Box (box.com a/k/a box.net) remote collection of resources


The following guide presents simple scenario how to WebID ACL protect a remote mounted Box (box.com a/k/a box.net) resources collection: 

   1 [[ODSGenerateX509Certificate][Generate an X.509 WebID Certificate ]], for ex. for this WebID:
<verbatim>
http://id.myopenlink.net/dataspace/person/demo#this
</verbatim>
   1. On a remote Virtuoso Briefcase instance [[OdsBriefcaseBoxNet][Mediate access to data stored on BoxNet]]. For ex. in folder named "MyBoxNet":
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl1.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl1.png" width="600px" /></a>%BR%%BR%
   1 Click "Edit":
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl2.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl2.png" width="600px" /></a>%BR%%BR%
   1 Go to "Sharing": 
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl3.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl3.png" width="600px" /></a>%BR%%BR%
   1 Set WebID ACL by adding the WebID from above:
<verbatim>   
http://id.myopenlink.net/dataspace/person/demo#this 
</verbatim>
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl4.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl4.png" width="600px" /></a>%BR%%BR%
   1 Export from your Browser Certificates store the generated certificate and its private key for ex. into file with name demo.pfx.
   1 Go to your local ODS Data Space instance at http://host:port/ods
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl5.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl5.png" width="600px" /></a>%BR%%BR%
   1 Click Sign In and enter your user's credentials:
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl6.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl6.png" width="600px" /></a>%BR%%BR%
   1 Go to Edit Profile -> Security -> Generate & Private Key:
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl7.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl7.png" width="600px" /></a>%BR%%BR%
   1 Import the private key from the generated from above certificate, for ex. under name: "MyKey":
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl8.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl8.png" width="600px" /></a>%BR%%BR%
   1 Go to Briefcase:
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl9.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl9.png" width="600px" /></a>%BR%%BR% 
   1 Click "Create new folder":
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl10.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl10.png" width="600px" /></a>%BR%%BR%
   1 In the presented form:
      * Specify name, for ex: "RemoteBoxNet";
      * Specify type "WebDAV":
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl11.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl11.png" width="600px" /></a>%BR%%BR%
   1 Go to the "WebDAV" tab:
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl12.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl12.png" width="600px" /></a>%BR%%BR%
   1 In the presented form enter respectively:
      * Set "Activity manager (on/off)" to on;
      * For "WebDAV path" enter the WebDAV path to the mounted from above remote collection <code> https://host:port/DAV/home/user-name/folder-name/ </code>;
      * Set "Authentication type" to "WebID";
      * Set in "User's Key" the imported private key "MyKey" from above;
      * Set "Graph name" or leave the default suggested value;
      * Set "Sponger" to on and specify which Extractor and Meta Cartridges to be included when sponging:
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl13.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl13.png" width="600px" /></a>%BR%%BR%
   1 Click "Create". 
   1 The WebDAV folder should be created:
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl14.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl14.png" width="600px" /></a>%BR%%BR%
   1. Click the link name of the freshly created "RemoteBoxNet" folder.
   1 Should be presented the content of the remote mounted BoxNet collection:
%BR%%BR%<a href="%ATTACHURLPATH%/bnetacl15.png" target="_blank"><img src="%ATTACHURLPATH%/bnetacl15.png" width="600px" /></a>%BR%%BR%
